# [부록] MVVM 패턴이란?
* Back로직, front의 마크업 & 데이터 표현단을 분리하기 위한 구조
* 전통적인 mvc패턴에서 파생
* 화면 앞단의 동작관련 로직과 뒷단의 DB데이터 처리및 서버 로직을 분리,  
뒷 단에서 넘어온 데이터를 Model에 담아 View로 넘겨주는 중간 단계


* view -- viewModel -- model
  - view -- viewModel : dataBinding 
  - model : business Logic & data
